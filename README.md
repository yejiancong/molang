# 《进步清单》
18年7月初因兴趣建立此项目，有幸得到大家关注，万分荣幸。现在把他迁移到github上，以方便更多人参与进来，目的为了提升技术能力。

## 撰写方法
### 认领内容
首先, 请在issues中建立一个新的issue, 说明你认领的内容, 和预计完成时间. 然后就可以Fork本项目进行编辑, 等你完成后, 发起一个Pull Request即可.
### 文件命名
每个章节建立一个md文件，如第1章的第1节，则建立**1.1.md**。

## 格式规范
请参看已有章节的规范, 要注意的是, 每个章节在底部都需要有一个links节, 包含"目录", "上一节", "下一节"的链接

## 编译方法
定期同步到tech-upgrade
